## 摘要
训练API调用的大型语言模型（LLM）代理需要大量高质量轨迹。然而，大规模收集此类数据通常需要具有可执行API和真实、预填充后端数据库的完全实现环境，这成为可扩展性的主要瓶颈。为了克服这一点，我们提出了一种无环境合成数据生成方法，该方法利用LLM作为即时数字世界模型。仅给定API规范，我们的方法生成模拟代理与有状态环境之间交互的轨迹。具体来说，LLM首先生成可通过提供的API解决的各种任务。然后，教师代理迭代地解决每个任务，而LLM模拟器根据任务上下文和模拟历史生成连贯的合成API响应。最后，LLM评判员过滤轨迹以确保所得数据集的质量。我们在具有挑战性的AppWorld和OfficeBench基准测试上评估了我们的方法，这些基准包括信息检索和状态更改任务。在我们的合成数据上微调模型产生了显著的性能提升，表明无需任何可执行环境即可为API调用代理生成有效的监督。我们的结果证明了基于LLM的API模拟作为一种实用、可扩展的解决方案，可用于跨不同API生态系统训练代理。

## Abstract
Training API-calling large language model (LLM) agents demands massive amounts of high-quality trajectories. However, collecting such data at scale typically requires fully implemented environments with executable APIs and realistic, pre-populated backend databases, creating a major bottleneck for scalability. To overcome this, we propose an environment-free synthetic data generation approach that leverages LLMs as on-the-fly digital world models. Given only API specifications, our method generates trajectories mimicking interactions between an agent and a stateful environment. Specifically, an LLM first generates diverse tasks solvable with the provided APIs. A teacher agent then iteratively solves each task while an LLM simulator generates coherent synthetic API responses conditioned on the task context and simulation history. Finally, an LLM judge filters the trajectories to ensure the quality of the resulting dataset. We evaluate our approach on the challenging AppWorld and OfficeBench benchmarks, which include both information-retrieval and state-changing tasks. Fine-tuning models on our synthetic data yields significant performance gains, demonstrating that effective supervision for API-calling agents can be generated without any executable environment. Our results establish LLM-based API simulation as a practical, scalable solution for training agents across diverse API ecosystems.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：提出ESAT（Environment-Free Synthetic Agentic Trajectory generation）管道，完全依赖API规格（名字、用途、输入/输出模式）生成多步代理轨迹，无需真实执行环境。
- **三个主要阶段**：
  1. **任务合成（Task Synthesis）**：
     - 基于API规格，由LLM（GLM-4.7-FP8）生成多样化且可求解的用户任务。
     - 采用**桶式生成**：将任务空间划分为组合网格（难度、动作类型、任务焦点、应用数量、每应用API数），确保平衡覆盖。
     - 使用**逆频率采样**：优先选择使用频率低的应用和API，保证覆盖均匀。
     - 任务经过LLM评判过滤（确保格式正确、可解），然后由LLM改写为简洁的意图级描述，再经二次评判。
  2. **轨迹合成（Trajectory Synthesis）**：
     - 教师代理（GLM-5.1-FP8）与LLM模拟器（同一模型）进行逐步骤交互。代理发出API调用，模拟器根据任务上下文、交互历史、虚拟用户信息、当前时间等条件，产生符合模式的状态一致响应。
     - 模拟器遵循四条核心规则：数据一致性（状态跨调用保持）、数据多样性（混合相关和不相关项）、数据真实性（满足字段约束）、模式合规。
     - 包含质量检查：输入验证、输出模式验证、语义正确性检验（由LLM评判），失败则重试固定次数后终止。
     - 轨迹终止条件：代理调用特殊完成任务API、达到步骤上限、使用无关应用、模拟失败。
  3. **轨迹过滤（Trajectory Filtering）**：
     - 由LLM评判（Gemini-3.1-Pro）多次评估轨迹的正确性和完整性，仅保留多数正面判决的轨迹。

## 6. 主要结论与发现

- **ESAT合成数据有效**：在AppWorld上，仅用ESAT-S52（52个独立于AppWorld的合成应用）微调即取得显著提升（最高47%），超越真实环境数据AWT微调的效果（Qwen3-4B：ESAT-S52 51.8% vs AWT 46.7%）。
- **环境无关性验证**：使用合成应用生成的轨迹能泛化到未见API和状态分布，说明数据具有鲁棒性。
- **超越大型模型**：4B~8B参数模型微调后性能接近或超越GPT-4o和Nemotron-3-120B；Qwen3.5-27B接近教师模型GLM-5.1-FP8。

- **对比现有合成数据集**：ToolAlpaca、ToolACE、Nemotron在AppWorld和OfficeBench上表现差或仅微弱提升，而ESAT带来巨大增益。

## 7. 优点

- **方法创新**：首次提出完全依赖API规格的无环境合成数据生成，消除了数据收集的基础设施瓶颈。
- **状态一致性**：模拟器通过上下文感知和历史维护实现状态连贯，支持写入型API和长步骤轨迹。
- **覆盖均衡**：采用逆频率采样和桶式生成，确保应用和API的均匀覆盖，避免分布偏倚。
- **质量保证**：多轮过滤（任务评判、模式验证、语义评判、轨迹评判）确保数据高质量，人类验证高一致性。
- **可扩展性**：仅需API规格即可为新域生成数据，便于迁移；合成应用集扩展了多样性。
- **实验扎实**：涵盖多模型、多基准、多对比、多次采样和充分消融，结果稳健。

## 8. 不足与局限
