## 摘要
传统机器人编程具有挑战性：它需要协调多模态感知、管理物理接触动力学，并处理多种配置和执行故障。我们提出了ASPIRE（通过迭代机器人探索的智能体技能编程），这是一个持续学习系统，能够以代码即策略范式自主编写和改进机器人控制程序，同时将经验累积到可复用的技能库中。ASPIRE发现的技能可跨任务、仿真与现实环境以及不同实体形态持续存在。它在一个开放循环中运行，包含三个组件：（1）闭环机器人执行引擎，可暴露细粒度的多模态轨迹，实现自主故障诊断、修复合成与验证；（2）持续扩展的技能库，将验证后的修复方法提炼为可复用、可迁移的知识；（3）进化搜索，可生成多样化的任务序列和控制程序，以超越单轨迹的改进探索。在扰动下，ASPIRE在LIBERO-Pro操控任务上比先前方法提升高达77%，在Robosuite双臂递送任务上提升72%，在BEHAVIOR-1K长时域家务任务上提升32%。其积累的技能库还能实现零样本泛化到未见过的长时域任务：在LIBERO-Pro Long上，ASPIRE的成功率为31%，而先前方法尽管使用了测试时推理与重试，成功率仅为4%。最后，仿真中发现的技能为仿真到现实的迁移提供了初步证据，显著减少了不同实体形态和机器人API下的实际机器人编程工作量。

## Abstract
Traditional robot programming is challenging: it requires orchestrating multimodal perception, managing physical contact dynamics, and handling diverse configurations and execution failures. We introduce ASPIRE (Agentic Skill Programming through Iterative Robot Exploration), a continual learning system that autonomously writes and refines robot control programs in a code-as-policy paradigm while compounding experience into a reusable skill library. ASPIRE discovers skills that persist across tasks, simulation and real-world settings, and embodiments. It operates in an open-ended loop with three components: (1) a closed-loop robot execution engine that exposes fine-grained multimodal traces, enabling autonomous failure diagnosis, repair synthesis, and validation; (2) a continually expanding skill library that distills validated fixes into reusable, transferable knowledge; and (3) evolutionary search that generates diverse task sequences and control programs to explore beyond single-trajectory refinement. ASPIRE surpasses prior methods by up to 77% on LIBERO-Pro manipulation under perturbation, 72% on Robosuite bimanual handover, and 32% on BEHAVIOR-1K long-horizon household tasks. Its accumulated library also enables zero-shot generalization to unseen long-horizon tasks: on LIBERO-Pro Long, ASPIRE achieves 31% success versus 4% for prior methods despite their use of test-time reasoning and retries. Finally, simulation-discovered skills provide initial evidence of sim-to-real transfer, substantially reducing real-robot programming effort across different embodiments and robot APIs.

## 1. 核心问题与整体含义（研究动机与背景）

传统机器人编程高度复杂，需要协调多模态感知、物理接触动力学、环境配置变化以及执行故障处理。现有的编码‑即‑策略（code‑as‑policy）方法虽然可以通过自然语言模型生成机器人控制程序，但存在两大根本局限：  
- **反馈粒度粗**：执行环境只提供任务级别的成功/失败信号，无法定位故障的根源（是感知、规划、抓取还是协调问题）。  
- **经验不累积**：完成一个任务后，发现的修复策略被丢弃，不能跨任务复用。人类工程师则会在调试过程中积累可迁移的修复知识（如多角度靠近、避碰策略等）。  

**ASPIRE** 的目标是构建一个能自主编写、调试并持续改进机器人控制程序的系统，同时将验证通过的修复模式累积为可复用的技能库，从而实现跨任务、跨环境、甚至跨实体形态的迁移学习。

## 2. 方法论：核心思想、关键技术细节

### 核心思想  
ASPIRE 采用“协调器‑执行器”架构，以代码即策略范式运行在一个开放式的学习循环中：  
1. **闭环机器人执行引擎**：为每个感知、规划、抓取、控制调用记录细粒度多模态轨迹（RGB关键帧、覆盖图、抓取候选、碰撞反馈等），使智能体能自主定位故障、合成修复并验证。  
2. **不断增长的技能库**：将验证通过的修复模式蒸馏为可复用的、可检索的上下文指导（含故障特征、何时应用、修复策略及代码草图）。  
3. **进化搜索**：在每次迭代中生成多样化的候选程序，基于执行结果和剩余故障选择优胜者，避免陷入局部修复循环。

### 关键技术细节  
- **执行引擎**：记录每个 API 调用的输入/输出、返回状态和关键帧；智能体不会看到完整视频帧，只关注故障涉及的原始调用证据。  
- **技能库**：技能由协调器从执行器报告的“发现”中提取，仅将通过调试验证且可泛化的模式加入库中。技能涵盖定位、导航、运动基元、物体抓取、场景理解和调试工作流等类别。  
- **进化搜索**（算法1）：初始程序 → 执行并评分 → 选 top-3 种群 → 结合技能库生成 K 个新候选 → 执行并选择最优 → 重复直到阈值或预算耗尽 → 在验证集上最终评估 → 提取有效模式加入技能库。

## 7. 优点

- **开放性知识积累**：技能库不是静态预设，而是从自主调试经验中归纳生成，类别丰富（定位、导航、抓取等）。  
- **细粒度调试能力**：执行引擎提供原始级多模态轨迹，使智能体能像人类工程师一样定位根本原因。  
- **跨实体形态零样本迁移**：仿真技能无需专门调整即可指导不同 API 的真实机器人编程，极具实用价值。  
- **进化搜索防止局部最优**：在单一修复循环上增加了探索多样性。  
- **实验评估全面**：同时包含仿真和真实迁移，消融实验清晰。

## 8. 不足与局限

- **非完全自主的真实世界部署**：真实机器人仍需要人工重置、安全监控、成功检测等环节，未形成闭环的终身学习。  
- **依赖高性能 LLM**：当前使用闭源前沿模型（Claude Opus 4.6、Codex GPT‑5.5），未测试较弱模型的可行性；计算成本高，LLM 调用和仿真滚动消耗大量资源。  
- **受限于预设 API**：系统只能表达已封装的原语，若任务需要新的传感或控制能力则需人工扩展。  
- **技能库管理未成熟**：随着库增长，可能出现过时、冗余、误导性条目，导致零样本迁移非单调；需要更稳健的检索、剪枝和重新验证机制。  
- **实验覆盖有限**：仅评估了少数仿真到真实的迁移案例，且真实任务仅3个，泛化能力仍需进一步验证。
