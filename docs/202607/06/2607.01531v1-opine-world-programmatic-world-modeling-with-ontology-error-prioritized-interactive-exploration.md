## 摘要
学习环境如何从交互中行为是构建适应不熟悉任务的智能体的核心。用深度网络学习的世界模型灵活但数据饥渴，且在其训练分布之外迁移能力差。程序综合的世界模型，由LLM以源代码形式编写并通过反例引导的归纳综合（CEGIS）改进，则数据高效且可重用，但它们主要在有给定对象词汇的结构化状态世界上展示，单个程序搜索无法扩展到像素渲染的环境，因为其对象结构必须灵活假设。我们提出OPINE-World，一个LLM智能体，从交互中在线学习以对象为中心的程序化世界模型。OPINE-World在假设和测试的循环中耦合两个协作智能体，一个在环境中行动，一个通过重放验证和基于模型的规划以代码综合模型，并通过我们称为本体错误的贝叶斯对象类型充足性度量引导探索。我们在ARC-AGI-3上评估OPINE-World，一个技能获取效率基准，其中对象词汇、目标和动作语义被隐藏。OPINE-World在没有每游戏训练的情况下解决了25个游戏中的20个，并达到了78.4的动作效率分数，与人类基线相比。

## Abstract
Learning how an environment behaves from interaction is central to building agents that adapt to unfamiliar tasks. World models learned with deep networks are flexible but data-hungry and transfer poorly beyond their training distribution. Program-synthesized world models, written as source code by LLMs and refined through counterexample-guided inductive synthesis (CEGIS), are instead data-efficient and reusable, yet they have been demonstrated mainly on structured-state worlds with a given object vocabulary, and a single program search does not scale to pixel-rendered environments whose object structure must be hypothesized flexibly. We introduce OPINE-World, an LLM agent that learns an object-centric programmatic world model online from interaction. OPINE-World couples two cooperating agents in a loop of hypothesis and test, one acting in the environment and one synthesizing the model in code with replay verification and model-based planning, and it steers exploration with a Bayesian measure of object-type adequacy we call ontology error. We evaluate OPINE-World on ARC-AGI-3, a benchmark for skill-acquisition efficiency in which the object vocabulary, the goal, and the action semantics are withheld. OPINE-World solves 20 of 25 games without per-game training and reaches an action-efficiency score of 78.4 against the human baseline.

## 2. 提出的方法论

### 核心思想
- **双智能体假设-测试循环**：耦合两个LLM智能体，一个负责在环境中行动并收集经验，另一个负责综合并修复对象中心的世界模型代码。
- **对象中心世界模型**：模型按对象类型（τ）组织，每个类型共享一个响应规则 fτ，结合局部上下文 u 预测每个对象的变化。
- **精确重放验证**：候选程序必须完全重现缓冲区中所有观察到的转移（状态属性完全一致），拒绝任何不一致或依赖隐藏状态的模型。
- **反例引导综合**：当模型在真实交互中产生错误预测时，记录为反例，触发综合者重写程序。
- **贝叶斯本体错误度量（ηt）**：基于狄利克雷后验计算每个对象类型的不确定性和各效果行的不确定性，通过噪声或组合成每个对象的ηit，聚合后引导探索优先关注当前类型解释不好的对象。
- **规划器**：模型通过验证且至少一个关卡被通过后，综合者为该模型编写规划器（有界前向搜索），在真实环境中逐步执行规划，每一步与预测对比，预测失败则作为反例返回。

### 关键技术细节
- **对象状态表示**：帧 → 抽象函数 α 提取对象集合，每个对象带有匹配键 κ、类型 τ、属性 v（位置、可见性、旋转、小像素模式）。
- **效果签名**：配对前后对象，记录哪些属性改变（如只改变x、同时改变x和像素等），丢弃具体值，形成类别签名。
- **局部效果表**：按 (类型, 动作, 局部上下文) 为行，统计各效果签名的次数，用对称狄利克雷先验估计行内分布。
- **类型不确定性**：通过候选类型下的似然计算每个对象类型的后验分布，归一化熵作为 U_type。
- **本体错误**：每对象 ηit = 1 - (1 - U_type)(1 - U_row)，聚合为全缓冲区的平均 ηt。

### 算法流程（文字说明）
1. 目标导向智能体在真实环境中执行动作，记录转移 (s, a, s') 到共享缓冲区。
2. 当当前模型预测错误或积累足够反例后，世界模型智能体基于缓冲区重写模型代码。
3. 模型通过精确重放验证后，若已有通关记录，则规划器在模型上搜索行动序列。
4. 规划步骤在真实环境中依次验证，预测不符则中断并记录反例。
5. 周期性运行批评者，挑衅模型的弱泛化能力，推动综合者探索更优假设。
6. 本体错误 ηt 持续计算，引导行动智能体探索高不确定性对象和上下文。

- **可解释可重用**：世界模型为可读的Python代码，可检查、可修复，且模型可复用于共享机制的任务。
- **双重验证机制**：精确重放验证保证模型与所有观测一致；反例引导修复严格逐步改进模型。
- **探索引导**：贝叶斯本体错误度量廉价且有效，优先探索模型当前解释不足的领域。
- **双智能体分工**：行动与模型分离，避免单一智能体兼任的认知负担，提升复杂任务性能（参考已有多智能体LLM工作）。

## 8. 不足与局限

- **可观察-马尔可夫性假设**：假设状态和动作完全决定下一状态，对于隐藏状态或随机环境，精确重放测试会拒绝正确模型，本体误差无法归零，方法适用性受限。
- **感知推断**：对象提取器由LLM推断，像素级别的对象分割和跨帧配对可能不准确，导致类型划分错误。
- **规划器规模**：采用有界前向搜索，在分支因子高的游戏中可能无法在预算内找到规划，从而退回到探索模式。
- **单次运行**：每游戏只运行一次，缺乏方差估计，结果可能会因随机性而变化。
- **缺少消融实验**：未系统评估每个组件对总体性能的贡献，无法确定各机制的实际重要性。
- **LLM 基础模型不同**：baseline1使用GPT-5.5，OPINE-World使用Claude Opus 4.8，两种模型不同，比较结果可能受基础模型能力差异影响。
- **延迟窗口和停滞守卫**：这些启发式参数未公开优化过程，可能对性能有影响。
- **未扩展到随机、部分可观察、连续动作空间**：方法目前限定于确定性、离散、可观察的简化环境。
