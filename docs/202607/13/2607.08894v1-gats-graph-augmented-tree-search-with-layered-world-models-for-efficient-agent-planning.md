## 摘要
大语言模型智能体在多步规划任务中展现潜力，但现有方法（如LATS和ReAct）在规划期间严重依赖LLM推理，导致高计算成本和随机行为。我们提出GATS（图增强树搜索），一种将基于UCB1的系统性树搜索与分层世界模型相结合的规划框架，在消除推理期间LLM调用的同时实现更优规划性能。我们的三层世界模型整合了：(L1) 精确符号动作匹配；(L2) 从执行日志中学习的统计信息；(L3) 对未知动作的LLM预测。在具有分支路径和死胡同的合成规划任务中，GATS达到100%成功率，而LATS为92%，ReAct为64%。在涵盖12个挑战场景（包括编码工作流、网页导航和长周期任务）的综合压力测试中，GATS保持100%成功率，而LATS降至88.9%，ReAct降至23.9%。GATS在规划期间每个任务需要零次LLM调用（LATS每个任务37次），并产生确定性规划，运行间方差为零。我们的结果表明，基于学习的世界模型的系统性搜索在智能体规划中显著优于LLM引导的探索。

## Abstract
Large Language Model (LLM) agents have shown promise in multi-step planning tasks, but existing approaches like LATS (Language Agent Tree Search) and ReAct rely heavily on LLM inference during planning, leading to high computational costs and stochastic behavior. We present \textbf{GATS} (Graph-Augmented Tree Search), a planning framework that combines systematic UCB1-based tree search with a layered world model to eliminate LLM calls during inference while achieving superior planning performance. Our three-layer world model integrates: (L1) exact symbolic action matching, (L2) statistics learned from execution logs, and (L3) LLM-based prediction for unknown actions. On synthetic planning tasks with branching paths and dead-ends, GATS achieves \textbf{100\% success rate} compared to 92 % for LATS and 64\% for ReAct. On a comprehensive stress test spanning 12 challenging scenarios -- including coding workflows, web navigation, and long-horizon tasks -- GATS maintains \textbf{100\% success} while LATS drops to 88.9 % and ReAct to 23.9%. GATS requires \textbf{zero LLM calls per task} during planning (vs. 37 per task for LATS) and produces deterministic plans with zero variance across runs. Our results demonstrate that systematic search with learned world models can substantially outperform LLM-guided exploration for agent planning.

## 2. 方法论：核心思想、关键技术细节
### 核心思想
将**世界模型**与**LLM**解耦：构建一个**三层世界模型**，在规划时优先使用低成本、确定性的前两层（符号匹配、统计学习），仅在遇到真正未知动作时才调用LLM（L3），且结果缓存复用。同时，使用**UCB1算法**在状态-转移图上进行系统性的搜索（而非LLM随机采样），保证探索与利用的平衡。

### 关键技术细节
- **问题形式化**：状态空间由命题集合表示，动作有前提条件和增删效果。目标是找到从初始状态到目标状态的动作序列。
- **三层世界模型**：
  - **L1 (符号层)**：对已知动作使用STRIPS风格的精确前提-效果匹配。若前提满足，返回确定的下一个状态（置信度1.0）；否则返回原状态（0.0）。成本几乎为零。
  - **L2 (统计层)**：从执行日志中学习动作的统计效果。维护动作→效果计数，返回出现最频繁的效果，置信度 = min(计数/10, 1.0)。成本很低。
  - **L3 (生成层)**：对未知动作调用LLM预测下一个状态，置信度设为0.5，结果缓存以避免重复调用。成本最高。
- **层选择规则**：依次查询L1、L2、L3，直到获得足够高置信度的预测（阈值τ1, τ2）。LLM调用次数由**新颖动作-状态对的数量**决定，而非规划步数。
- **状态-转移图记忆**：构建有向图G，节点代表规范化的状态key，边代表动作→预测下一状态。**相同状态合并**（类似置换表），不同路径到达同一状态共享节点，复用搜索统计信息。图在规划步骤间持久保留，而非每次丢弃。
- **UCB1选择**：在每个状态，将可行动作视为臂，使用UCB1公式选择动作：
  \[
  a_t = \arg\max_a \left[ Q(s,a) + c \sqrt{\frac{2\ln N(s)}{N(s,a)}} \right]
  \]
  未访问动作具有无限UCB值，保证每个动作至少被评估一次。
- **状态价值估计**：使用BFS计算到目标的最短预测距离d_G(s, G)，价值函数为：
  \[
  \text{StateValue}(s) = \begin{cases} \frac{1}{\alpha d_G(s,G)+1}, & \text{如果目标可达} \\ 0, & \text{否则} \end{cases}
  \]
  其中α=10。价值加权世界模型置信度：`v = p · StateValue(s') - λ(1-p)`。
- **规划循环**：从当前状态运行GANTSearch（有限预算b），选择最佳动作，执行并移动到预测状态，重复直至达到目标或达到最大步数。
