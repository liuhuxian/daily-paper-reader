## 摘要
我们提出了一种基于可供性识别和动作效果预测的操控规划系统。该系统以视觉形式推理可能的未来，并通过将预测结果与运行时设定的文本目标进行多模态目标匹配来评估候选规划。即使对象被遮挡，或初始描述符无法在未来状态中识别它们，通过在预测中跟踪目标文本中命名的对象位置，系统仍能生成动作规划。我们进一步扩展系统，添加了一个图像转换模块，用于将具有不同形状和视觉外观的真实世界状态图像转换为一致的视觉外观，从而在物理机器人设置中促进操控规划。我们分别评估了系统各模块的性能，并在仿真和硬件上针对一系列具有挑战性的任务展示了集成系统的操控规划能力。

## Abstract
We present a manipulation planning system based on affordance recognition and action effect prediction. The system reasons through possible futures in visual form, and evaluates candidate plans by agreement of predicted outcomes with text-based goals set at run-time, using a multi-modal goal-matching module. Positions of objects named in the goal text are tracked through predictions even when occluded, making it possible to generate action plans even when objects become occluded, or when their initial descriptors cease to identify them in future states. We further expand the system with an image conversion module for translating real-world state images with objects of varied shapes and visual appearances into a consistent visual appearance, to facilitate manipulation planning in a physical robot setup. We evaluate performance of the system's modules in isolation and demonstrate the integrated system's manipulation planning capabilities on a set of challenging tasks in both simulation and on hardware.

### 方法论：核心思想、关键技术细节

**核心思想**：系统通过“**视觉前向推理**”的方式规划。它迭代进行“**识别可供性 → 预测未来状态 → 评估状态与目标匹配度**”的过程，最终选择能最大化目标达成程度的动作序列。

**关键技术细节**：

1.  **系统架构**：由五个核心模块组成，协同工作。
    *   **目标解释模块 (Goal-Interpretation Module, GIM)**：
        *   **输入**：初始状态图像 + 用户的文本目标（如“[小球]在[大立方体]的右边”）。
        *   **功能**：利用SAM（Segment Anything Model）半自动地为文本中提到的对象生成**对象掩码**（Object Masks），覆盖所有RGBD状态图像的三个额外通道（RGBDMMM格式）。掩码值1表示可见，0.5表示被遮挡但存在，0表示不存在。这使得系统能在未来状态中持续追踪这些对象，即使它们被遮挡。
    *   **可供性识别模块 (Affordance Recognition Module, ARM)**：
        *   **功能**：给定一个RGBD状态图像，检测当前场景中存在的所有可供性（如抓取、放置、推、插入等）及其参数（3D坐标和夹爪角度）。
        *   **实现**：基于修改的ScaledYOLOv4网络，输出可供性姿态而非物体的边界框。
    *   **效果预测模块 (Effect Prediction Module, EPM)**：
        *   **功能**：给定当前状态和一个可供性，预测执行该动作后的下一个状态（RGBDMMM图像）。
        *   **实现**：一个编码器-预测器-解码器结构的神经网络。
        *   **关键技术**：
            *   **双视角处理**：为了利用动作效果的旋转/平移不变性，图像被分成“机器人视角”（固定）和“桌子视角”（根据当前动作参数进行平移和旋转，使动作中心对齐）。两个视角的图像被拼接后输入网络。
            *   **正交投影**：仿真和转换后的图像均使用正交投影，以消除相机平移引入的视差变化，简化图像变换。
            *   **记忆痕迹**：预测器会生成一个记忆痕迹向量，在预测多步动作时传递，以保留可能延迟生效的效果信息。
    *   **目标匹配模块 (Goal-Matching Module, GMM)**：
        *   **功能**：量化一个预测的未来状态与文本目标的匹配程度。
        *   **实现**：一个多模态神经网络（基于CLIP架构改进）。
        *   **输入**：目标图像的掩码通道（`masks`）+ 标准化后的目标文本（`goal`）。
        *   **输出**：一个[0,1]的实数值，表示一致性。
        *   **处理多句目标**：分别计算每句的一致性，取最小值作为最终分。
    *   **图像转换模块 (Image Conversion Module, ICM)**：
        *   **功能**：将真实世界的RGBD图像转换为仿真环境的视觉风格（Sim-to-Real泛化）。
        *   **实现**：基于Swin-UNet，将4通道RGBD真实图像转换为仿真风格的RGBD图像。
        *   **训练数据生成**：使用ControlNet + Stable Diffusion，以仿真图像中的物体深度图为条件，生成外观多样但形状匹配的真实感物体图像，再合成完整的场景图像，解决了真实-仿真图像配对数据难获取的问题。

2.  **规划流程**：
    *   **第一步**：获取初始状态，通过ICM（若硬件实验）和GIM处理，得到RGBDMMM状态和标准化文本目标。
    *   **循环迭代**：
        1.  ARM识别当前状态下的可行动作列表。
        2.  对于每个候选动作，EPM预测下一个状态。
        3.  GMM评估所有预测状态与目标的匹配度，选出最佳动作。
        4.  对于选出的最佳动作，将其预测状态作为新的当前状态，重复步骤1-3，直到达到预设的搜索深度。
    *   **剪枝**：对预测结果相近的动作进行剪枝，减少搜索树的规模。
    *   **输出**：选择目标匹配度最高的状态序列所对应的动作序列作为最终规划。

### 优点：亮点

*   **方法设计**：
    1.  **基于目标的规划**：用户只需描述“要达成什么”（结果），而非“要做什么”（步骤），更自然直观。
    2.  **视觉前向推理**：将规划问题转化为对未来视觉状态的评估，利用预测网络处理细粒度的物理效果，避免了符号方法的抽象化信息损失。
    3.  **遮挡处理**：通过对象掩码（RGBDMMM）清晰地对目标对象进行显式追踪，并允许在遮挡状态下（掩码值0.5）进行判断，这是处理真实世界遮挡问题的优雅方案。
    4.  **通用性强的域适应**：利用条件扩散模型合成多样的训练数据来训练图像转换网络，解决了Sim-to-Real中数据匮乏的难题，且使系统能够泛化到不同颜色和纹理的物体。
*   **实验设计**：
    1.  **模块化解耦**：策略上先对各模块独立评估，再对整个系统进行集成评估，便于定位问题。

### 不足与局限

1.  **Sim-to-Real泛化瓶颈**：ICM在堆叠物体场景、物体偏心放置等未在训练数据中充分覆盖的情况下表现不佳，导致硬件实验成功率下降严重（从9/10到3/5的任务成功率超过0）。
2.  **模块集成误差**：各模块独立训练，虽然也采用了数据增强，但无法完美模拟前序模块的输出特性（如ICM转换后的伪影、EPM预测的模糊等），导致误差累积。
3.  **规划效率**：系统规划时间过长（从数秒到数分钟），主要归因于GMM计算代价高、双框架（PyTorch + JAX）导致的数据转换开销大，限制了其在需要实时响应的场景中的应用。
4.  **物体识别依赖**：GIM目前仍需要半自动的用户交互（用SAM点选物体）来生成初始掩码，未能实现完全自主的物体实例识别。
5.  **对象类别泛化局限**：虽然ICM能适应不同视觉外观，但系统训练和实验的对象仅限于四种预设类别（球、立方体、杯、盘），对新类别物体的泛化能力未知。
6.  **规划搜索空间**：目前的暴力搜索算法效率低下，作者自己也提到，未来可以结合符号推理进行更高效的规划。
