## 摘要
多任务机器人操作策略难以从示范中学习，因为传统上单个网络必须根据语言和视觉上下文，从多模态示范分布中定性不同的动作模式中进行选择。错误的模式选择意味着执行错误的任务或在场景中不可行的动作。将连续动作标记化为学习到的离散码本，在表示层面分离这些模式，为多任务学习提供了结构优势。我们提出VQActFlow，一种多任务操作策略，它将动作块标记化并通过变分流匹配生成码序列。VQActFlow在整个生成过程中维持对动作模式的显式偏好。推理时的引导作用于这种偏好以引导模式承诺。我们通过语言条件下的无分类器引导（将策略引导至指示的动作模式）以及一个学习到的码本评价器（提供互补的可行性信号）来实现这一点。我们在三个平台上评估VQActFlow：LIBERO模拟基准、执行全身抓取的Unitree G1人形机器人，以及执行接触丰富任务的ALOHA-style双臂平台。在这些基准测试中，VQActFlow优于连续和离散基线方法。

## Abstract
Multi-task robot manipulation policies are challenging to learn from demonstration because traditionally a single network must select among qualitatively different action modes from a multimodal demonstration distribution, conditioned on language and visual context. A wrong mode selection means executing the wrong task or an action infeasible in the scene. Tokenizing continuous actions into a learned discrete codebook separates these modes at the representation level, offering structural advantages for multi-task learning. We propose VQActFlow, a multi-task manipulation policy that tokenizes action chunks and generates code sequences via Variational Flow Matching. VQActFlow maintains an explicit preference over action modes throughout generation. Inference-time guidance acts on this preference to steer mode commitment. We instantiate this with classifier-free guidance over language conditioning, which steers the policy toward the instructed action mode, and a learned codebook critic that supplies a complementary feasibility signal. We evaluate VQActFlow on three platforms: the LIBERO simulation benchmarks, a Unitree G1 humanoid performing whole-body pick-and-place, and an ALOHA-style bimanual platform performing contact-rich tasks. Across these benchmarks, VQActFlow outperforms both continuous and discrete baselines.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：多任务机器人操作策略需要从多模态演示数据中学习，根据语言指令和视觉观察选择正确的“动作模式”（action mode）。传统方法（如扩散策略、流匹配）在连续动作空间生成，无法显式表示离散的动作模式，因此当观测模糊时，错误的模式选择会导致执行错误任务或不可行的动作。
- **动机**：将连续动作离散化为码本（VQ-VAE）可以在表征层面分离不同的动作模式，为多任务学习提供结构优势。但现有离散策略（如Discrete Policy）仅在最后一步通过最近邻查找恢复离散索引，生成过程中缺乏对模式承诺的显式表示。
- **整体含义**：VQActFlow 提出在生成过程中**持续维持对各动作模式的显式偏好**（categorical distribution），并通过推理时引导（语言CFG + 码本评价器）来主动引导模式选择，从而解决多任务场景下的模式歧义问题。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 2.1 核心思想
- **两阶段框架**：第一阶段：使用 VQ-VAE 将连续动作块（action chunk）令牌化为离散码本索引序列；第二阶段：训练一个**变分流匹配（VFM）策略**，生成从高斯噪声到码本嵌入的流，并输出每个位置对码本索引的**对数几率（logits）**，从而在每一步都维持对各动作模式的偏好。
- **推理时引导**：通过两种方式作用于logits空间：
  1. **无分类器引导（CFG）**：在语言条件上线性外推logits，强化语言指令对应的动作模式。
  2. **码本评价器（Codebook Critic）**：一个小型Transformer，通过对比学习（InfoNCE）训练，输出场景/语言条件共同下的可行性分数，其梯度用于调整策略的类别分布，引导策略选择**在当前场景中可行的**动作模式。

### 2.2 关键技术细节
1. **VQ-VAE 动作令牌化**

   - 每个 z(l) 量化为最近邻码本 e_k，得到索引序列 {k*_l}，解码器 D 重构动作块。
   - 训练后冻结，为VFM提供监督目标。

2. **变分流匹配策略（VFM）**

3. **CFG 作用于logits**

## 7. 优点

- **方法创新**：
  - 提出在动作生成过程中显式维持对离散动作模式的偏好，并利用该偏好的logits空间进行引导，比以往仅在最后一层离散化更自然。

## 8. 不足与局限

- **VQ-VAE瓶颈**：信息损失本质上界定了策略性能上限；更先进的令牌化方法（如连续量化、残差VQ）可能有更大潜力，本文未探索。
- **数据集规模较小**：硬件任务每类仅75-140个演示，在更复杂场景（如长期任务、细粒度灵巧操作）上可能不足。
- **CFG权重和评价器强度需手动调参**：虽然CFG权重扫描了合理范围，但在实际部署中需要针对具体任务微调，增加使用成本。
