## 摘要
现代视觉-语言-动作（VLA）模型通常难以泛化到新的设置，例如改变的摄像头视角或机器人形态，因为它们通常仅基于当前观测和语言指令进行条件设定。通过忽略作为变量的底层系统配置，这些模型隐含地假设了训练期间遇到的固定执行上下文，导致任何新环境都需要数据密集型的微调。在这项工作中，我们引入了上下文世界建模（ICWM），这是一个将系统辨识视为上下文自适应问题的框架。ICWM使机器人策略能够从短期的自生成、与任务无关的交互历史中自主推断出关键的系统变量。与使用示范来指定要执行何种任务的传统上下文学习不同，ICWM利用上下文窗口来理解系统如何运行。通过在任务执行前处理这些交互，模型隐式地捕获了当前系统的世界动态，从而无需参数更新即可适应新的配置。在仿真和真实世界机器人平台上的广泛实验表明，ICWM在新型摄像头视角上显著优于标准的VLA基线模型。

## Abstract
Modern Vision-Language-Action (VLA) models often fail to generalize to novel setups, such as altered camera viewpoints or robot morphologies, because they are typically conditioned only on current observations and language instructions. By ignoring the underlying system configuration as a variable, these models implicitly assume a fixed execution context encountered during training, necessitating data-intensive fine-tuning for any new environment. In this work, we introduce In-Context World Modeling (ICWM), a framework that treats system identification as an in-context adaptation problem. ICWM enables robot policies to autonomously infer essential system variables from a short history of self-generated, task-agnostic interactions. Unlike traditional In-Context Learning that uses demonstrations to specify what task to perform, ICWM leverages the context window to understand how the system operates. By processing these interactions before task execution, the model implicitly captures the world dynamics of the current system, enabling adaptation to novel configurations without parameter updates. Extensive experiments in simulation and on real-world robot platforms demonstrate that ICWM significantly outperforms standard VLA baselines on novel camera viewpoints.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：现代视觉-语言-动作（VLA）模型（如RT-2、OpenVLA、π0等）在训练时通常仅基于当前观测和语言指令进行条件设定，忽略了底层系统配置（如摄像头视角、机器人运动学形态等）这一变量。这导致模型隐式假设了一个固定的执行上下文，当部署环境（如新的摄像头视角、机器人形态或语义场景）与训练分布不同时，性能显著下降，需要数据密集的微调才能适应。

- **核心问题**：VLA对系统配置的泛化失败本质上是**系统辨识问题**——策略在测试时缺乏对当前配置ψ的显式知识。现有解决方法（如场景特定微调、人类示范注入）耗时且违背通用部署目标。

- **整体含义**：人类在操作不熟悉控制映射时会先进行随机探索来建立内部世界模型，然后才执行任务。受此启发，论文提出**上下文世界建模（ICWM）**，将系统辨识重构为一个**上下文自适应问题**，使机器人能够通过自生成的、任务无关的短交互历史隐式恢复当前系统动力学，从而无需参数更新即可适应新配置。

## 2. 方法论：核心思想、关键技术细节、公式/算法流程

### 2.1 核心思想
- **重新定义上下文的作用**：不同于传统上下文学习（ICL）将上下文用于行为指定（“做什么”），ICWM将上下文用于**系统辨识**（“系统如何运作”）。
- **利用交互上下文进行隐式世界建模**：在任务执行前，机器人执行一段短时随机探索运动，记录视觉-动作-下一观测三元组作为上下文。模型通过处理这些三元组隐式捕获当前系统的输入-输出映射（如摄像头-动作对应关系），从而在后续任务中生成与当前物理设置一致的动作。

### 2.2 关键技术细节
- **理论动机**：论文证明交互上下文`T = (o_{0:t}, a_{1:t})`携带关于系统配置ψ的信息严格多于单个观测`o_0`（Proposition 1），因为多步观测-动作对能够揭示隐藏的状态（包括时间不变的系统配置ψ）。
- **架构设计**：采用参数高效方式，让**上下文处理与VLA主干共享参数**。模型基于预训练VLM（Qwen2.5-VL-3B）和动作分词器（FAST），将交互上下文clip（N个三元组）与任务查询（当前观测+语言指令）拼接作为Transformer输入。上下文生成的隐藏状态`Ψ(T)`隐式编码了世界动力学。
- **训练流程**：在包含多种系统配置（多视角、多形态）的训练数据上，对每个训练样本**前置N个随机采样的任务无关交互clip**（这些clip来自同一轨迹/视角下的探索段）。模型学习从上下文中提取配置信息以正确预测任务动作，损失为负对数似然`L = -log πθ(at | Ψ(T), ot, l)`。
- **推理流程**：
  1. **探测阶段**：机器人执行N个随机目标姿态的探索运动（无任务干扰），记录三元组`(o_s, a, o_e)`，形成上下文T。
  2. **执行阶段**：将T与任务查询一起输入模型，Transformer先对T进行注意力处理构建配置感知的隐藏状态，再生成任务动作。无需梯度更新或任务特定示范。

### 2.3 公式与算法（文字说明）
- 标准VLA：`πθ(at | ot, l)` 忽略ψ → 泛化失败。
- 理想策略：`π*θ(at | ot, l, ψ)`。
- ICWM：`at ∼ πθ(at | Ψ(T), ot, l)`，其中`Ψ(T)`从交互上下文隐式推断ψ。
- 训练公式：`L = -log πθ(at | Ψ(T), ot, l)`，T来自同一配置下的随机探索段。

## 6. 主要结论与发现

1. **视角泛化是根本挑战**：标准VLA在新视角上性能显著下降（LIBERO上MV从74.5%→19.8%），ICWM通过上下文自适应显著缓解，OOD成功率平均提升13.0%。
2. **隐式辨识优于显式指定**：ICWM比显式输入真实摄像头角度的EXP高出9.5%，说明上下文比孤立的几何信息能提供更丰富的动力学知识。
3. **对长程任务增益最大**：在LIBERO-Long上，ICWM比MV提升26.3%（OOD），因为长程任务放大了视角偏移导致的微小空间误差。
4. **真实世界有效性**：UR5e上，ICWM在6个新视角的平均成功率比MV提升71%（从17%→29%左右），验证了实际部署价值。
5. **上下文必须一致**：错误上下文（错误视角）导致性能低于无上下文，表明模型确实从上下文中提取配置信息，而非忽略。
6. **形态与语义泛化**：ICWM在干扰物、新纹理和不同连杆长度上均保持超越MV的增益，说明其普适性。
7. **探测策略鲁棒**：各种简单随机策略均有效，但不同方向组合提供最佳信息。

- **无监督/自监督适应**：探测完全自生成、任务无关，无需人类示范、奖励信号或梯度更新，极适合实际部署。

## 8. 不足与局限

- **某些视角仍困难**：如135°视角在所有方法上成功率都很低（<3%），ICWM提升有限，说明存在视角特有感知瓶颈（遮挡/视野不足）。
- **探测过程依赖随机探索质量**：在真实世界需确保安全随机运动（避障、不干扰任务对象），对复杂场景可能需要更精细的碰撞检测；本文使用固定工作空间边界，但并非完全自动。
- **上下文长度开销**：虽然推理延迟可接受，但每次推理需处理2N+N个额外token，长期运行时KV缓存管理会带来内存压力（本文未详细分析）。
- **实验覆盖有限**：仅测试摄像头视角、语义和简单形态变化，未涉及其他类型OOD（如光照、运动学奇异点、传感器故障等）。
- **跨具身泛化未充分验证**：WindowX平台仅测试变连杆长度，更复杂的跨机器人结构或自由度变化未考察。
- **无开放式环境测试**：真实实验在固定桌子、已知物体集上进行，对完全未知场景的泛化能力未知。
