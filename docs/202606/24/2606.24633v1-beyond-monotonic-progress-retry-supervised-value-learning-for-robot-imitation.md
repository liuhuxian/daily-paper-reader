## 摘要
人类用于机器人模仿学习的演示常包含错误和纠正行为，例如不精确的抓取、物体错位、不稳定的接触以及重复尝试。尽管这些片段通常被视为有噪声或次优数据，但它们提供了关于执行何时偏离期望路径以及如何恢复任务可行性的宝贵证据。然而，现有的奖励和价值模型往往依赖单调进展假设，这捕捉了粗略的任务进展，但可能忽略了不完美演示中的局部执行错误和纠正行为。在这项工作中，我们提出了ReTVL（重试监督的价值学习）框架，通过利用重试事件作为稀疏监督，从混合质量的机器人演示中学习对错误敏感的价值函数。ReTVL通过结合全局进展校准与由稀疏标注的重试关键点诱导的局部成对偏好学习，捕捉错误周围的局部退化-恢复结构。然后，学习到的价值模型用于对下游行为克隆的演示片段进行重新加权，减少有害执行错误的影响，同时保留有用的纠正行为。在真实机器人操作任务上的实验表明，ReTVL相比基于进展的基线产生了更细粒度的价值估计，并改善了从不完美演示中的模仿学习。

## Abstract
Human demonstrations for robot imitation learning often contain mistakes and corrective behaviors, such as imprecise grasps, object misalignment, unstable contact, and repeated attempts. While these segments are commonly treated as noisy or suboptimal data, they provide valuable evidence about when execution deviates from a desirable path and how task feasibility can be restored. However, existing reward and value models often rely on monotonic progress assumptions, which capture coarse task advancement but may overlook local execution errors and corrective behaviors in imperfect demonstrations. In this work, we propose ReTVL (ReTry-Supervised Value Learning), a framework for learning mistake-sensitive value functions from mixed-quality robot demonstrations by leveraging retry events as sparse supervision. ReTVL captures the local degradation-and-recovery structure around mistakes by combining global progress calibration with local pairwise preference learning induced by sparsely annotated retry keypoints. The learned value model is then used to reweight demonstration chunks for downstream behavior cloning, reducing the influence of harmful execution errors while preserving useful corrective behaviors. Experiments on real-robot manipulation tasks show that ReTVL produces more fine-grained value estimates than progress-based baselines and improves imitation learning from imperfect demonstrations.

## 2. 方法论

### 核心思想
- 利用**重试关键点（retry keypoint）**——即纠正行为开始的时刻—作为局部监督锚点。这些关键点仅需稀疏标注（标注成本低），且比完整的错误段更可靠。
- 结合**全局进展校准**与**局部成对偏好学习**，使价值函数既能反映任务整体进展，又能捕捉错误附近的局部价值下降与反弹。

### 关键技术细节
- **问题设置**：价值函数 \(V_\theta(h_t, \ell)\) 基于图像历史 \(h_t\) 和语言指令 \(\ell\) 预测标量进度（0~1）。使用**分布值函数**（预测离散进度箱上的分类分布）增加稳定性。
- **全局进展监督**：对非重试区域的帧，使用绝对进度目标（成功轨迹中 \(v_t^* = t/T\)，失败轨迹终端为0），用交叉熵损失 \(L_{\text{abs}}\) 训练。
- **成对偏好学习**：
  - 围绕每个重试关键点 \(r\)，定义三个区域：**pre-retry窗口**（\(W_{\text{pre}}\)）、**near-retry窗口**（\(W_{\text{near}}\)）、**post-retry窗口**（\(W_{\text{post}}\)）。
  - 构造偏好对 \((h^+, h^-)\)，其中 \(h^+\) 预期值更高（pre侧远离重试的帧、post侧远离重试的帧），\(h^-\) 预期值更低（near侧帧）。
  - 使用**软窗口加权**：根据帧与重试关键点的距离赋予指数衰减权重 \(w = \exp(-d/\tau_w)\)，减少边界不确定性带来的噪声。
  - 偏好损失 \(L_{\text{pref}} = -\mathbb{E}[w \log \sigma((V^+ - V^-)/T_{\text{pref}})]\)。
- **最终损失**：\(L_{\text{value}} = \lambda_{\text{abs}} L_{\text{abs}} + \lambda_{\text{pref}} L_{\text{pref}}\)。
- **价值指导的行为克隆**：根据价值改进量 \(r_t = V(h_{t+\Delta_a}) - V(h_t)\) 计算每个动作块的权重 \(\alpha_t\)，然后进行加权流匹配行为克隆 \(L_{\text{wBC}}\)，抑制低改进或负改进的块，强调高改进的块。

## 3. 实验设计

### 任务与数据集

  - Stack Blocks（精细对齐三块积木）

  - Open Drawer（铰接物体互动，打开、放物体、关闭）
- **数据**：每个任务30条标注轨迹训练价值模型，20条保留轨迹用于价值评估；200条额外轨迹（除Pick up Spoon用80条）用于下游策略学习。
- **标注**：仅标注重试关键点（纠正开始时刻），不标注完整错误段。

### 基线方法
- **TOPRevard**：用VLM的“true” token概率作为进度分数。
- **Robometer**：基于大规模机器人数据训练的通用的奖励模型，用轨迹回放和偏好学习。
- **RECAP-Value**：论文实现的RECAP中的价值组件，用蒙特卡洛步进成功目标学习进度价值。

- **全局指标**：Value-Order Correlation (VOC) 在干净成功轨迹上的单调性；Success/Fail Detection (S/F Det.) 终端值区分成功/失败。

- RECAP-BC（用RECAP-Value加权）

## 6. 主要结论与发现

1. **价值质量**：ReTVL在全局指标上保持竞争力（VOC 0.987，S/F Det. 1.000），同时在局部错误敏感性上显著优于基线（Drop AUC 0.797 vs 基线最高0.372；Pre > Retry 0.740 vs 基线最高0.329）。
2. **策略改进**：ReTVL-BC在四个任务上的平均成功率为80%，比Standard BC（41.25%）高38.75%，比RECAP-BC（62.50%）高17.50%。
3. **有害动作抑制**：ReTVL将坏动作的平均权重降至0.11，远低于RECAP-BC的0.54。
4. **消融启示**：偏好损失是错误敏感性的关键，绝对校准提供全局锚点，软窗口稳定训练。

## 7. 优点

- **创新性**：首次将重试事件作为稀疏监督用于价值学习，揭示了错误-恢复结构的价值，避免了单调进展假设的局限性。
- **设计合理**：软窗口加权缓解了边界模糊问题；分布值函数增加稳定性；偏好损失直接学习局部相对顺序而非硬编码下降。
- **实用性**：重试关键点标注简单（每轨迹约0.6分钟），成本低，且标注一致性高（平均误差0.19秒）。
- **下游效果显著**：不仅提升成功率，还大量抑制有害动作而保留有用纠正。

## 8. 不足与局限

- **假设限制**：假设重试事件附近存在退化-恢复模式，但现实中的纠正行为可能包括探索性重试、不成功的纠正等，该假设可能不覆盖所有情况。
- **应用范围**：仅用于离线行为克隆重加权，未探索在线强化学习或闭环策略改进。
- **实验规模**：仅在4个真实任务上验证，任务类型和演示风格有限。未使用大规模标准数据集（如BridgeData、RT-1数据集），泛化能力待检验。
- **外部因素**：基线Robometer和TOPRevard的预训练数据可能包含与任务相关的先验，而ReTVL仅在较小任务数据集上微调，对比公平性需注意。
